# Таблица позиций: поиск, фильтр пустых, тултипы заголовков

Дата: 2026-07-11

> Небольшое UI-улучшение `PositionsTable`/`PortfolioTab`. Три независимые
> фичи, объединены в одну спеку из-за общего места изменений и малого
> размера каждой.

## 1. Поиск по тикеру/названию

- Текстовый инпут в новой строке контролов над `PositionsTable`.
- Фильтрует `calculated` по подстроке (регистронезависимо) в `p.ticker`
  ИЛИ `p.shortName`.
- Пустая строка поиска — фильтр не применяется (показаны все позиции).

## 2. Чекбокс «Скрывать пустые позиции»

- В той же строке контролов, рядом с поиском.
- «Пустая позиция» = `p.sharesOwned === 0`. Статус (`in_index` /
  `out_of_index`) не учитывается — только количество акций.
- При включённом чекбоксе такие позиции убираются из отображаемого
  списка.

## 3. Комбинирование фильтров

- Оба фильтра — AND: сначала поиск, затем скрытие пустых (порядок не
  важен, результат одинаков).
- Реализация: новый `useMemo` в `PortfolioTab.tsx`, вычисляющий
  `filteredPositions` из `calculated` — передаётся в `PositionsTable`
  вместо `calculated`.
- Сводка портфеля (`portfolioValue`, `avgCompliance` — блок
  `.portfolio-summary`) продолжает считаться из **не**фильтрованного
  `calculated`. Фильтрация — чисто отображение таблицы, на итоговые
  цифры не влияет.

## 4. Персистентность в localStorage

- Значения поиска и чекбокса переживают перезагрузку страницы.
- Ключи: `portfolio.search`, `portfolio.hideEmpty`.
- Инициализация состояния читает `localStorage` при маунте (лениво,
  через функцию-инициализатор `useState`), запись — через `useEffect`
  на изменение значения.
- Реализуется инлайн в `PortfolioTab.tsx`, без отдельного модуля —
  единственная точка использования, обобщать незачем.

## 5. Тултипы в заголовках таблицы

- Значок `?` рядом с текстом заголовка — только у расчётных/неочевидных
  колонок:

  | Колонка       | Текст тултипа                                                      |
  |---------------|---------------------------------------------------------------------|
  | Коэф-т        | Множитель к весу в индексе при расчёте целевой доли                |
  | Цель          | Целевая доля = вес в индексе × коэффициент                         |
  | Факт. доля    | Текущая доля позиции в стоимости портфеля, %                       |
  | Соответствие  | Факт. доля ÷ Цель (1.0 = точное совпадение)                        |
  | Доход         | Дивиденд на акцию × количество акций                                |

  (Формулы соответствуют `domain/calculations.ts`: `computeTargetAllocation`,
  `computeActualShare`, `computeCompliance`, `computeIncome`.)
- Механизм — чистый CSS, без библиотек: `<th>` получает
  `position: relative`, внутри — `<span class="th-hint" data-tooltip="...">?</span>`;
  `.th-hint[data-tooltip]::after` показывается по `:hover`/`:focus`
  (для доступности с клавиатуры — `tabIndex={0}` на span).
- Остальные 10 колонок (Тикер, Название, Вес в индексе, Цена, Лотность,
  Сектор, Дивиденд, Статус, Куплено, Стоимость) — без тултипов, их смысл
  очевиден без пояснения.

## 6. Затрагиваемые файлы

- `webapp/src/components/PortfolioTab.tsx` — состояние `search`/`hideEmpty`,
  localStorage-синхронизация, `filteredPositions`, строка контролов.
- `webapp/src/components/PositionsTable.tsx` — разметка тултипов в
  заголовках.
- `webapp/src/styles.css` — стили строки контролов и CSS-тултипа.

Изменений в `domain/`, `file/schema.ts`, `iss/` — нет. Чисто UI-слой,
существующая расчётная логика не трогается.
