## 1. Фикс тултипов в заголовках таблицы (регрессия)

Баг: текст тултипа `?`-иконок в `<thead>` не появляется.

Причина (`webapp/src/styles.css:375-413`): `.th-hint[data-tooltip]::after`
использует `position: absolute`, но ни `.th-hint`, ни родительский `<th>`
не имеют `position: relative` — блок позиционируется относительно
ближайшего позиционированного предка выше по дереву, а не рядом с
иконкой. Плюс `.table-scroll` (`:330-334`) задаёт только
`overflow-x: auto`, из-за чего `overflow-y` вычисляется в `auto` и тоже
клипает контент — вертикально всплывающий тултип обрезается.

Фикс:
- `.th-hint` получает `position: relative`.
- `.table-scroll` получает `overflow-y: visible` (горизонтальный скролл
  остаётся, вертикальный клип убирается).

## 2. Дашборд: наибольший избыток / наибольшая недостача

В `Dashboard.tsx`, третий и четвёртый стат-блок (после «Общая стоимость»,
«Среднее соответствие»):

- Отклонение позиции, ₽: `(actualShare − targetAllocation) * portfolioValue / 100`.
  Считается по всем позициям с ненулевым `targetAllocation`
  (те же, что участвуют в `avgCompliance`).
- «Наибольший избыток» — позиция с максимальным положительным
  отклонением. «Наибольшая недостача» — с минимальным (наиболее
  отрицательным).
- Формат: `<тикер> <знак>₽<значение>` (напр. `SBER +₽12 340`).
- Пусто/нет позиций с targetAllocation → «—» (как у `avgCompliance`).
- Реализация: новый `useMemo` рядом с `useCalculatedPositions()` в
  `Dashboard.tsx` либо в `useCalculatedPositions.ts` — вычисляет
  `{ ticker, deviationRub }[]`, берёт min/max.

## 4. Колонка «Див доходность, %»

Новая расчётная колонка сразу после «Дивиденд» (`:39`).

- Формула: `dividend / price * 100`. Оба поля уже есть на позиции
  (live-данные из ISS). При `price === 0` или `dividend` отсутствует —
  показывать «—».
- Добавляется в `buildCalculatedPositions.ts` как новое поле
  `CalculatedPosition.dividendYield: number | null`, по аналогии с
  `income`.
- Тултип не нужен (формула очевидна из соседства с «Дивиденд» и «Цена»).

## 5. Колонки «Акций купить» и «Купить на сумму»

Две новые колонки, обе — сразу после «Куплено» (`:42`), в таком
порядке: Куплено → Акций купить → Купить на сумму → Цель → …

- `targetShares = round(targetAllocation / 100 * portfolioValue / price)`
  (лотность не учитывается — целые штуки).
- `sharesToBuy = targetShares − sharesOwned`. Положительное — сколько
  докупить; отрицательное — излишек, сколько продать (знак минус,
  как есть, без спец-форматирования типа скобок).
- `buyAmountRub = sharesToBuy * price`. Тоже может быть отрицательным
  (продажа) — значение как есть.
- `targetAllocation === null` (не в индексе) или `price === 0` →
  обе колонки «—».
- Оба поля — новые в `CalculatedPosition`: `sharesToBuy: number | null`,
  `buyAmountRub: number | null`. Считаются в `buildCalculatedPositions.ts`
  рядом с `income`.
- Тултип у «Акций купить»: «Целое число акций до целевой доли; минус —
  продать». У «Купить на сумму» — без тултипа (произведение соседних
  колонок).

## 6. Подсветка колонок «Коэф-т» и «Куплено»

Фон ячеек (`<td>` с `<input>`) — лёгкий акцентный оттенок
(напр. `var(--accent-dim)`, уже используется для hover-строки — взять
близкий, но отличимый тон, либо новая CSS-переменная), чтобы визуально
отделить два редактируемых пользователем поля от расчётных колонок.
Применяется через CSS-класс на `<td>` (напр. `.td-editable`), не на
весь столбец через `nth-child` — проще при добавлении/перестановке
колонок.

## 7. Колонка «Сектор» — в конец

`PositionsTable.tsx`: заголовок и ячейка «Сектор» (`:38`, `:39` в теле)
переносятся в конец строки — после «Доход», последней колонкой.

## 8. Колонка «Статус» — иконка первой колонкой, без заголовка

- В `<thead>`: первая `<th>` — пустая (без текста), либо `<th></th>`
  для сохранения структуры таблицы.
- В `<tbody>`: первая `<td>` содержит только индикатор (текущий
  `.status-dot`/`.status-dot--in`, без текстового лейбла
  `STATUS_LABELS`), перед тикером.
- Убирается текущая колонка «Статус» (была между «Дивиденд» и
  «Коэф-т», `:40`) — не дублируется.
