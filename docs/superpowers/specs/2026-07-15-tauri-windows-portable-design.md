# Tauri Windows Portable — дизайн

**Дата:** 2026-07-15  
**Статус:** утверждён

## Цель

Добавить portable-версию приложения для Windows на Tauri 2, чтобы синхронизация
с Т-Банком выполнялась локально и не блокировалась браузерной политикой CORS.
Пользователь не поднимает прокси, не устанавливает сервер и не передаёт токен
третьей стороне.

Текущая веб-версия на GitHub Pages сохраняется. Finam продолжает работать в
браузере, а синхронизация с Т-Банком становится доступна только в Windows-версии.

## Принятые ограничения

- Первая desktop-версия поддерживает только Windows.
- Основной артефакт распространения — portable `.exe`, без установщика.
- Автоматическое обновление не входит в первую версию.
- Windows-сборка на первом этапе не подписывается сертификатом издателя. На
  странице релиза нужно предупредить о возможном сообщении SmartScreen.
- Бизнес-логика брокерской синхронизации остаётся в TypeScript.
- Внешний или локально запускаемый пользователем proxy-сервер не используется.

## Архитектура

В `webapp/src-tauri` добавляется минимальная оболочка Tauri 2. Rust-часть не
реализует брокерские операции, не хранит токены и не предоставляет универсальную
proxy-команду. Она подключает официальный Tauri HTTP plugin.

Сетевой слой TypeScript получает узкую абстракцию транспорта:

- браузерный транспорт вызывает глобальный `fetch`;
- Tauri-транспорт вызывает `fetch` из `@tauri-apps/plugin-http`;
- клиент Т-Банка выбирает Tauri-транспорт только в desktop-среде;
- клиенты Finam и MOEX продолжают использовать текущий браузерный транспорт.

Определение среды централизовано в одной функции `isTauriRuntime()`. Компоненты и
брокерские клиенты не проверяют глобальные признаки Tauri самостоятельно.
Tauri-модули загружаются динамически, чтобы обычная Vite-сборка для GitHub Pages
не зависела от наличия desktop runtime.

Tauri capability разрешает HTTP plugin обращаться только к адресу:

```text
https://invest-public-api.tbank.ru/**
```

Произвольные адреса из UI в нативный HTTP-клиент не передаются. Content Security
Policy запрещает загрузку произвольных удалённых скриптов.

## Поток синхронизации

Существующий пользовательский сценарий не меняется:

1. Пользователь вводит токен Т-Банка.
2. Приложение получает список счетов через нативный HTTP-транспорт.
3. Пользователь выбирает счёт.
4. Приложение получает портфель и разрешает идентификаторы инструментов в тикеры.
5. Пользователь видит предварительный просмотр изменений.
6. Изменения применяются только после подтверждения.

Токен продолжает шифроваться существующим механизмом и храниться в
`portfolio.json`. Для синхронизации он расшифровывается в памяти frontend-процесса
и передаётся локальному Tauri HTTP plugin. Токен не сохраняется в Rust-части, не
попадает в логи и не включается в тексты ошибок.

## Поведение веб-версии

В браузере Т-Банк остаётся виден в списке брокеров, но добавление нового
подключения и запуск синхронизации недоступны. Интерфейс показывает отдельное
пояснение: «Синхронизация с Т-Банком доступна в приложении для Windows» и ссылку
на страницу GitHub Release с portable-версией.

Ограничение применяется только к Т-Банку. Finam в веб-версии работает как раньше.

Если в браузере открыт `portfolio.json` с ранее созданным подключением Т-Банка,
подключение и связанные данные сохраняются. Повторное сохранение файла не удаляет
и не изменяет их; недоступно только выполнение сетевой синхронизации.

## Ошибки

Клиент преобразует технические сбои в пользовательские категории:

- попытка синхронизации Т-Банка в браузере — требование Windows-версии;
- HTTP `401`/`403` — неверный токен или недостаточные права;
- HTTP `429` — превышен лимит запросов;
- HTTP `5xx` или отсутствие сети — временная недоступность API;
- неожиданный JSON — несовместимый контракт ответа Т-Банка.

Заголовок `Authorization`, токен и полные тела ответов API не логируются.

## Сборка и распространение

Существующая сборка и публикация GitHub Pages остаётся самостоятельной. Для
desktop-версии добавляется отдельная команда Tauri build и отдельный Windows job
GitHub Actions. Job собирает release executable и прикладывает portable `.exe` к
GitHub Release.

Portable-приложение использует системный Microsoft Edge WebView2. Требование
WebView2 и ссылка на официальный установщик указываются в описании релиза на
случай его отсутствия в системе.

## Проверка

- Unit-тест выбора браузерного и Tauri-транспорта.
- Unit-тест клиента Т-Банка с внедрённым mock-транспортом.
- Компонентный тест: Т-Банк заблокирован в браузере и доступен в Tauri.
- Регрессионный тест: Finam доступен в обеих средах.
- Тест сохранения подключения Т-Банка при открытии и сохранении файла в браузере.
- Smoke-тест запуска Tauri-приложения на Windows.
- Ручной контрактный тест Т-Банка с реальным токеном, включаемый переменной
  окружения и не запускаемый в стандартном CI.
- Полный прогон существующих `test`, `typecheck`, `lint`, `build` и E2E.

## Не входит в объём

- Поддержка macOS и Linux.
- Установщики MSI/NSIS.
- Подпись Windows-кода сертификатом.
- Автоматическое обновление.
- Перенос брокерских адаптеров или шифрования токенов в Rust.
- Изменение транспорта Finam или MOEX.
