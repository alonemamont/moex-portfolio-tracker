# Портфель-трекер IMOEX: функциональная спека веб-приложения

Дата: 2026-07-10

> Эта спека описывает **только функциональность**. Технологии, архитектура,
> хостинг, фреймворки, БД/файловое хранилище на уровне реализации — предмет
> отдельной сессии (см. раздел "Вне скоупа этой спеки" внизу).

## Контекст и происхождение

Прототип функциональности — личная Excel-таблица (`MICEX Index 99,5_.xlsx`,
репозиторий `moex-portfolio-tracker`), которая отслеживает состав индекса
МосБиржи (IMOEX) и сравнивает с ним личный портфель. Изначально таблица
тянула тикеры/веса/цены/дивиденды через мёртвый внешний источник
(Google Sheets `IMPORTRANGE`). Разработан (см.
`2026-07-10-moex-live-data-design.md`) переход на live-данные через
MOEX ISS API — бесплатный, публичный, без ключа.

Веб-приложение — следующий шаг: та же функциональность, но не завязанная на
Excel/VBA, плюс история и графики. Экселевская спека остаётся источником
истины по бизнес-правилам (сопоставление тикеров, статус "в/вне индекса",
защита ручных полей) — они переносятся 1:1.

## Цель

Веб-приложение, которое:
1. Показывает актуальный состав индекса IMOEX с рыночными данными.
2. Позволяет вести личный портфель поверх этого состава (сколько куплено,
   с каким коэффициентом отклонения от индексного веса).
3. Считает соответствие портфеля индексу и фактическую стоимость портфеля.
4. Хранит историю цен/стоимости портфеля и строит графики.
5. Держит личные данные пользователя локально — без аккаунта, без сервера,
   хранящего чужие данные. Пользователь сам загружает свой файл, работает
   с ним, выгружает обратно.

## Функциональные возможности

### 1. Состав индекса и рыночные данные (live)

Источник — MOEX ISS API (тот же, что в Excel-спеке):

| Поле | Источник | Примечание |
|---|---|---|
| Тикер | ISS, состав индекса | ключ сопоставления строк |
| Название бумаги | ISS | |
| Вес в индексе (%) | ISS, состав индекса | 0, если бумага помечена "вне индекса" |
| Цена | ISS, `LAST`, фоллбэк `PREVPRICE` | нужна всегда, даже для бумаг вне индекса — для стоимости портфеля |
| Лотность | ISS | |
| Сектор | **ручной справочник** тикер→сектор | в ISS официального классификатора нет, ведётся пользователем |
| Дивиденд на акцию | ISS, история выплат, последнее значение | факт последней выплаты, не прогноз; 0 для бумаг без истории — не ошибка |
| Статус | вычисляется | "в индексе" / "вне индекса" |

Пагинация ISS может молчаливо обрезать список — при получении состава
индекса нужно запрашивать полный список, не ограничиваясь дефолтным
размером страницы.

### 2. Личные данные пользователя (ручные поля)

Для каждой позиции пользователь задаёт:
- **Коэффициент** — множитель отклонения личного веса позиции от
  индексного веса (дефолт 1.0 для новых позиций).
- **Количество купленных акций** — дефолт 0 для новых позиций.

Эти два поля — **исключительно ручные**. Обновление рыночных данных их
никогда не трогает, независимо от того, редактировал их пользователь
раньше или нет.

### 3. Обновление рыночных данных (сопоставление по тикеру)

Правила переноса 1:1 из Excel-дизайна:

- **Тикер есть в портфеле и в свежем составе индекса** → обновить
  название/вес/цену/лотность/дивиденд, статус = "в индексе".
- **Тикер есть в портфеле, но выпал из состава индекса** → строка не
  удаляется, статус = "вне индекса", вес обнуляется (не участвует в
  целевом распределении и в расчёте соответствия индексу), но цена и
  дивиденд всё равно обновляются (нужны для расчёта фактической стоимости
  и дохода позиции). Коэффициент и количество купленных акций не трогать.
- **Тикер новый в составе индекса, которого нет в портфеле** → добавляется
  новая позиция автоматически, коэффициент = 1.0, куплено = 0, статус =
  "в индексе".
- Сопоставление всегда по тикеру (регистронезависимо), не по порядку/номеру
  строки — состав индекса и порядок бумаг в нём может меняться.
- Повторное обновление подряд не создаёт дублей и не меняет ручные поля.

### 4. Расчёты

- **Целевое распределение** позиции — на основе веса в индексе и
  коэффициента (только для позиций "в индексе").
- **Соответствие индексу** — отклонение фактической доли позиции от
  целевой; считается **только** по позициям "в индексе". Деление на
  нулевой вес не должно приводить к ошибке — просто пустой/нулевой
  результат для таких строк.
- **Общая стоимость портфеля** — сумма (цена × количество купленных) по
  **всем** позициям, включая "вне индекса".
- **Доход по позиции** (дивиденды × количество купленных) — по всем
  позициям, включая "вне индекса".
- **Среднее соответствие индексу** (агрегат по всему портфелю) — считается
  только по позициям "в индексе", независимо от того, сколько всего бумаг
  в списке (размер списка не должен быть захардкожен нигде в расчётах).

### 5. Секторальный справочник

Ручная таблица тикер → сектор, редактируемая пользователем. Используется
для группировки/анализа портфеля по секторам. Не тянется из ISS (поле
пустое у эмитентов).

### 6. История и графики (новое, отсутствовало в Excel)

- При каждом обновлении рыночных данных сохраняется **снимок** (дата/время,
  цена каждой бумаги, стоимость портфеля, состав "в/вне индекса" на тот
  момент).
- Графики:
  - Цена отдельной бумаги во времени.
  - Общая стоимость портфеля во времени.
  - Соответствие индексу во времени (агрегат).
- Глубина истории и частота снимков — определяется частотой, с которой
  пользователь запускает обновление (нет фонового автообновления —
  приложение не предполагает постоянно работающий сервер, см. ниже).

### 7. Хранение личных данных: локальный файл, без аккаунта

Ключевое отличие от типичного веб-приложения: **никакого логина, никакого
серверного хранения чужих данных**. Пользователь:

1. Открывает приложение (локально или по ссылке).
2. Загружает свой файл с данными (позиции: тикер, коэффициент, количество
   купленных; плюс, если есть, накопленная история для графиков).
3. Работает — редактирует позиции, жмёт "обновить рыночные данные".
4. Выгружает файл обратно себе (перезаписывает исходный, содержащий
   актуальные позиции и обновлённую историю).

Приложение не хранит данные пользователя между сессиями на сервере —
единственный источник правды это файл, которым владеет пользователь.
Работает по аналогии с текущим циклом Excel (`positions.csv` +
"Загрузить"/"Сохранить"), но здесь **весь** рабочий набор данных (не
только позиции, но и накопленная история) — часть этого файла.

### 8. Обработка ошибок (перенос принципов из Excel-дизайна)

- Источник данных (MOEX ISS) недоступен — показать ошибку, не затирать
  существующие данные частичным/неполным обновлением ("всё или ничего").
- Тикер не торгуется/делистинг — цена не обновляется (остаётся последнее
  известное значение), статус = "вне индекса".
- Пустая история дивидендов — 0, не ошибка.
- Загрузка файла позиций с тикером, которого нет в текущем составе —
  предупреждение, не молчаливое отбрасывание.

## Пользовательские сценарии

1. **Первый запуск.** Пользователь открывает приложение без файла →
   получает пустой/шаблонный портфель на основе текущего состава индекса
   (все коэффициенты 1.0, куплено 0) → редактирует под себя → выгружает
   файл.
2. **Регулярное использование.** Пользователь загружает свой файл →
   жмёт "Обновить" → видит актуальные цены/веса/статусы и обновлённые
   графики → при необходимости меняет коэффициенты/количество купленных
   акций (например, после покупки/продажи) → выгружает файл обратно.
3. **Анализ.** Пользователь смотрит графики стоимости портфеля и
   соответствия индексу за период, смотрит разбивку по секторам.
4. **Индекс поменялся.** Бумага вышла из индекса — пользователь видит
   статус "вне индекса", понимает, что она больше не участвует в расчёте
   соответствия, но всё ещё учитывается в общей стоимости — решает,
   продавать или держать.

## Данные, переносимые из Excel-дизайна без изменений

- Полный список полей на бумагу (тикер, название, вес, цена, лотность,
  сектор, дивиденд, статус, коэффициент, куплено).
- Правила сопоставления по тикеру при обновлении (раздел 3 выше).
- Формула защиты ручных полей (коэффициент/куплено никогда не
  перезаписываются обновлением рыночных данных).
- Правило "вне индекса": не участвует в соответствии индексу, участвует в
  общей стоимости портфеля.
- Источник данных — MOEX ISS API (состав индекса, цены/лотность,
  история дивидендов).

## Вне скоупа этой спеки

Обсуждается в новой сессии перед написанием implementation plan:
- Технологический стек (фреймворк, язык, фронтенд/бэкенд).
- Формат и структура файла для хранения позиций+истории (CSV/JSON/др.).
- Способ загрузки/выгрузки файла в браузере (file picker, drag&drop,
  автосохранение и т.п.).
- Хостинг (локально/облако), необходимость бэкенда вообще (может быть
  чисто клиентское приложение, раз данных на сервере нет).
- Обращение к MOEX ISS напрямую из браузера vs через прокси
  (CORS-ограничения ISS не проверялись).
- UI/UX, вид графиков, конкретная библиотека для них.
- Тестирование/CI/деплой.
- Возможное добавление аккаунтов/мультиустройства в будущем (сейчас явно
  вне скоупа — выбрана модель "один файл, без логина").
