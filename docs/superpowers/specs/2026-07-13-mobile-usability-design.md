# Мобильная юзабилити

Дата: 2026-07-13

> Полный функциональный паритет с десктопом на мобильных экранах.
> Один брейкпоинт (`max-width: 600px`), mobile-first правки поверх
> текущего `styles.css` и компонентов — без отдельного мобильного
> клиента. Основной вызов — таблица позиций на 17 колонок, которая
> ниже брейкпоинта превращается в список карточек.

## 1. Breakpoint

Единственный брейкпоинт: `@media (max-width: 600px)`. Выше — текущая
десктопная раскладка без изменений. Планшеты в портретной ориентации
(≥600px) продолжают получать десктопный вид с горизонтальным скроллом
таблицы, как сейчас.

## 2. Шапка (`Header.tsx` + `.header`)

Ниже брейкпоинта:

- Раскладка: `IMOEX ▾` (select) слева, заголовок «Портфель-трекер» по
  центру (можно укоротить/скрыть текст, оставить только иконку/бренд),
  кнопка `⋮` справа.
- `⋮` открывает меню действий (Загрузить файл / Сохранить / Начать с
  пустого портфеля) — простой dropdown или bottom-sheet, использующий
  тот же набор обработчиков (`handleLoadClick`, `handleSaveClick`,
  `handleStartEmpty`), что и сейчас; логика не меняется, меняется
  только их визуальный контейнер.
- Скрытый `<input type="file">` не трогаем.

## 3. Dashboard-бар (`Dashboard.tsx` + `.dashboard`)

Ниже брейкпоинта: `.dashboard` — `overflow-x: auto`,
`flex-wrap: nowrap` (вместо переноса), горизонтальный скролл по 4
метрикам вместо схлопывания в несколько строк. Контент и `data-label`
псевдоэлементы не меняются.

## 4. Табы (`.tabs`)

Без структурных изменений. Ниже брейкпоинта увеличить `padding` кнопок
табов так, чтобы высота тап-зоны была ≥44px (сейчас `padding: 10px 2px`
даёт по факту заметно меньше).

## 5. Таблица позиций → карточки (`PositionsTable.tsx`)

Ниже брейкпоинта `<table className="positions-table">` заменяется на
список карточек. Новый компонент `PositionCard.tsx`, рендерится вместо
`<table>` при выполнении media query (через CSS + `matchMedia`/hook —
конкретный механизм переключения оставляется на усмотрение реализации,
главное чтобы разметка карточек и разметка таблицы не рендерились
одновременно).

Свёрнутая карточка показывает поля для беглого просмотра:

- статус-точка (`status-dot`)
- Тикер + короткое имя
- Цена
- Соответствие (`ComplianceGauge`)

По тапу на карточку — разворачивается остальные поля списком
`label: value` (Вес в индексе, Лотность, Дивиденд, Див доходность,
Коэф-т, Куплено, Акций купить, Купить на сумму, Цель, Факт. доля,
Стоимость, Доход, Сектор). Редактируемые поля (Коэф-т, Куплено)
остаются `<input type="number">`, вызывают те же
`onChangeCoefficient` / `onChangeSharesOwned`, что и сейчас — меняется
только контейнер, не логика.

Существующие фильтры/сортировка в `PortfolioTab.tsx` (`controls-row`,
`action-row`) продолжают работать над тем же массивом `positions`;
список карточек рендерится из того же отфильтрованного/отсортированного
массива, что сейчас передаётся в `<PositionsTable>`.

## 6. Модалки (`AddTickerModal`, `PairPositionsModal`, `SectorOverrideModal`)

Ниже брейкпоинта `.modal-backdrop`/`.modal` — full-screen: модалка
занимает весь экран (`position: fixed; inset: 0`) вместо центрированного
окна с `min-width: 320px`. Внутреннее содержимое (таблицы, чекбоксы,
инпуты) скроллится вертикально как есть. Кнопки в `.modal__actions` —
full-width, высота ≥44px.

## 7. Графики (`ChartsTab`, `HistoryLineChart`, `SectorDonutChart`)

Уже используют `<ResponsiveContainer width="100%">` — структурных
изменений не требуется. Ниже брейкпоинта: уменьшить `padding` у
`.history-chart`/`.sector-chart`, чуть увеличить `fontSize` у
`tick`/`Legend` для читаемости на маленьком экране.

## 8. Touch targets и инпуты (глобально, ниже брейкпоинта)

- `button`, `select`, инпуты внутри карточек позиций — `min-height: 44px`.
- `input[type="number"]`, `input[type="text"]`, `select` —
  `font-size: 16px` (меньший размер шрифта провоцирует авто-зум при
  фокусе в iOS Safari).

## Тестирование

- Ручная проверка в Chrome DevTools device toolbar (375px, 390px,
  414px) и на реальном телефоне: шапка/меню, dashboard-скролл, карточки
  позиций (сворачивание/разворачивание, редактирование Коэф-т/Куплено),
  full-screen модалки, графики.
- Существующие Vitest-тесты компонентов не завязаны на разметку
  `<table>` жёстко — при появлении `PositionCard.tsx` добавить
  отдельные unit-тесты на рендер полей и вызов
  `onChangeCoefficient`/`onChangeSharesOwned`, по аналогии с текущими
  тестами `PositionsTable`.
