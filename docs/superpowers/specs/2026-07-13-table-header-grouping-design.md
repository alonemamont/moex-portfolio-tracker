# Визуальная группировка двухуровневого заголовка таблицы позиций

Дата: 2026-07-13

> Доработка поверх `2026-07-13-table-modal-refinements-design.md`: текущий
> двухуровневый `<thead>` (группы «Куплено»/«Купить») визуально нечитаем —
> непонятно, каким заголовкам второго уровня соответствует заголовок
> первого уровня. Плюс два новых группирующих заголовка: «Вес %» и
> «Дивиденды».

## 1. Визуальное выделение группы (общий паттерн)

Проблема: заголовок группы (`th-group`, `colSpan={2|3}`, строка 1) сейчас
не связан визуально со своими заголовками второго уровня (строка 2) —
между ними проходит обычная `border-bottom`, такая же как между любыми
двумя случайными ячейками. Ничего не показывает, что они образуют одну
группу, и ничего не показывает её левую/правую границу.

Единый паттерн для любой группы (переиспользуется всеми тремя группами:
«Вес %», «Куплено», «Купить», «Дивиденды»):

- Фон группы: `var(--accent-dim)` — уже существующий токен (используется
  для hover-подсветки строк тела), переиспользуется здесь без добавления
  новой переменной.
- Заголовок группы (`th-group`, строка 1): `border-left`/`border-right:
  1px solid var(--accent)`, `border-bottom: none` (убирает
  разделительную линию между группой и её подколонками — обычная
  `.positions-table th { border-bottom }` подавляется явным `none`).
  Так как это одна `<th colSpan={N}>`, её левая/правая граница совпадает
  с истинными границами группы.
- Подколонки группы (строка 2): тот же фон `var(--accent-dim)`; только
  крайняя левая ячейка группы получает `border-left`, только крайняя
  правая — `border-right` (тем же цветом `var(--accent)`), достраивая
  рамку вниз. `border-bottom` у подколонок остаётся обычным (это нижняя
  граница всего `<thead>`).

Соседние группы (например «Куплено» и «Купить» стоят рядом) — их
соприкасающиеся `border-right`/`border-left` просто дают одну более
заметную вертикальную линию между группами, дополнительной обработки не
требуется.

`styles.css`: два новых модификатора на существующих ячейках —
`.th-group-start`/`.th-group-end` (для строки 2, отвечают за
крайние border) — плюс правки существующего правила `.positions-table
th.th-group` (убрать `border-bottom`, добавить `border-left`/
`border-right`, фон).

## 2. Новая группа «Вес %»

Между «Название» и «Цена», `colSpan={2}`:

- Заголовок 1 уровня: `Вес %`
- Заголовок 2 уровня: `в индексе` | `фактический`

Заменяет текущие независимые заголовки «Вес в индексе, %» и «Факт. вес,
%» (`headerWithHint`). Тултип с текста «Текущая доля позиции в стоимости
портфеля, %» переезжает на `фактический`. Данные в `<td>` не меняются —
меняется только текст и структура заголовка. Порядок колонок не
меняется (обе колонки уже стоят рядом).

## 3. Новая группа «Дивиденды»

Между «Соответствие» и «Сектор», `colSpan={3}`:

- Заголовок 1 уровня: `Дивиденды`
- Заголовок 2 уровня: `Размер ₽` | `Доходность %` | `Доход ₽`

Заменяет текущие независимые заголовки «Дивиденд», «Див доходность, %»,
«Доход» (`headerWithHint` на «Доход»). Тултип с текста «Дивиденд на
акцию × количество акций» переезжает на `Доход ₽`. Данные в `<td>` не
меняются. Порядок колонок не меняется (все три колонки уже стоят
рядом).

## Итоговая структура `<thead>`

Строка 1 (`rowSpan={2}` для обычных колонок, `colSpan={N}
className="th-group"` для групп):

```
статус | Тикер | Название | [Вес %] | Цена | Лотность | Коэф-т |
[Куплено] | [Купить] | Цель | Соответствие | [Дивиденды] | Сектор
```

Строка 2 (только под группами):

```
в индексе | фактический | Штук | Стоимость | Штук | На сумму |
Размер ₽ | Доходность % | Доход ₽
```

  `th-group-start`/`th-group-end` на подколонках.
- `webapp/src/styles.css` — правки `.positions-table th.th-group`,
