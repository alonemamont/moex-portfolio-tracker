# Вкладка «Транзакции»: дизайн

Дата: 2026-07-13

## Контекст и цель

Приложению нужна ручная история внешних денежных потоков портфеля: пополнений
брокерских счетов и выводов с них. На первом этапе транзакции хранятся и
отображаются с агрегатами, но не участвуют в расчёте доходности портфеля.

Данные остаются частью пользовательского `portfolio.json`, чтобы переноситься и
сохраняться вместе с остальными данными портфеля. Серверное или браузерное
хранилище для транзакций не вводится.

## Объём первой версии

- новая вкладка «Транзакции»;
- ручное добавление, редактирование и удаление пополнений и выводов;
- дата операции, сумма, валюта, опциональный брокерский счёт и комментарий;
- ручной справочник брокерских счетов;
- валюты RUB, USD и CNY;
- итоги пополнений, выводов и чистого потока отдельно по валютам;
- автоматическая сортировка по дате по убыванию;
- фильтр по валюте;
- адаптивное отображение списка на узких экранах.

В первую версию не входят расчёт доходности с поправкой на денежные потоки,
пересчёт валют и автоматический импорт операций от брокеров.

## Модель данных

В `PortfolioFile` добавляются два нормализованных массива:

```ts
interface BrokerAccount {
  id: string;
  name: string;
}

type TransactionType = "deposit" | "withdrawal";
type TransactionCurrency = "RUB" | "USD" | "CNY";

interface Transaction {
  id: string;
  type: TransactionType;
  amount: number;
  currency: TransactionCurrency;
  date: string;       // YYYY-MM-DD
  comment?: string;
  accountId?: string; // отсутствие означает «Общий портфель»
}

interface PortfolioFile {
  // существующие поля
  brokerAccounts: BrokerAccount[];
  transactions: Transaction[];
}
```

Транзакция ссылается на запись справочника по `accountId`. Встроенное в
транзакцию название счёта не хранится: переименование счёта должно сразу
отражаться во всех связанных операциях.

Старые файлы без новых полей загружаются с пустыми массивами. Версия формата
остаётся `1`, поскольку изменение обратно совместимо при чтении. При следующем
сохранении новые массивы записываются в файл.

Идентификаторы счетов и транзакций уникальны внутри соответствующих массивов.
Новые идентификаторы создаются средствами браузера. `accountId`, если указан,
должен ссылаться на существующий ручной счёт.

## Компоненты и границы

### Доменная логика транзакций

Чистые функции отвечают за создание, изменение и удаление транзакций,
сортировку, валютную фильтрацию и расчёт итогов. Они не зависят от React и не
обращаются к файловой системе.

### Доменная логика счетов

Отдельные функции добавляют, переименовывают и удаляют ручные счета. При
удалении счёта все транзакции с его `accountId` атомарно переводятся в «Общий
портфель» удалением ссылки.

### Пользовательский интерфейс

`TransactionsTab` получает и обновляет `PortfolioFile` через существующий
`PortfolioContext`. Компонент координирует панель итогов, валютный фильтр,
список и модальные окна, но не содержит расчётную логику.

Отдельные модальные компоненты отвечают за форму транзакции и управление
справочником счетов. Это позволяет изменять их независимо от списка.

### Файловая схема

Zod-схема задаёт значения по умолчанию для новых массивов и проверяет структуру
записей, допустимые значения типа и валюты, сумму, дату, уникальность
идентификаторов и ссылочную целостность `accountId`.

## Интерфейс вкладки

Вкладка «Транзакции» добавляется в существующую навигацию приложения.

Верхняя панель содержит:

- агрегаты «Пополнения», «Выводы» и «Чистый поток»;
- фильтр «Все валюты», RUB, USD, CNY;
- кнопку «Добавить транзакцию»;
- кнопку «Счета».

При фильтре «Все валюты» агрегаты выводятся отдельными строками по каждой
валюте, в которой есть операции. Суммы разных валют не складываются. При выборе
конкретной валюты и агрегаты, и список ограничиваются этой валютой.

Список всегда сортируется по `date` по убыванию. При одинаковой дате новые
операции показываются выше ранее добавленных: новая запись добавляется в начало
массива, а сортировка по дате остаётся стабильной. Редактирование даты
автоматически меняет положение операции.

На широком экране список отображается таблицей с колонками:

- дата;
- тип;
- сумма и валюта;
- счёт;
- комментарий;
- действия редактирования и удаления.

На узком экране каждая операция становится карточкой, чтобы чтение суммы и
использование действий не требовали горизонтальной прокрутки.

Если операций нет вообще или нет для выбранной валюты, вкладка показывает
соответствующее пустое состояние. В общем пустом состоянии доступна кнопка
добавления первой операции.

## Добавление и редактирование

Форма транзакции открывается в модальном окне и содержит:

- тип «Пополнение» или «Вывод»;
- положительную сумму;
- валюту RUB, USD или CNY;
- дату с текущей локальной датой по умолчанию;
- счёт: «Общий портфель» или запись ручного справочника;
- необязательный комментарий;
- действия «Сохранить» и «Отмена».

Будущие даты разрешены без отдельного статуса и сразу участвуют в агрегатах.
Редактирование использует ту же форму с исходными значениями. Удаление
транзакции требует подтверждения.

## Управление счетами

Модальное окно «Счета» показывает ручной справочник и позволяет:

- добавить счёт с обязательным названием;
- переименовать существующий счёт;
- удалить счёт после подтверждения.

При удалении предупреждение сообщает, сколько транзакций будет переведено в
«Общий портфель». Сами транзакции не удаляются.

Справочник ручных счетов независим от запланированных подключений к брокерским
API. Возможное объединение этих сущностей потребует отдельного дизайна.

## Валидация и нормализация

- сумма должна быть конечным числом больше нуля и иметь не более двух знаков
  после десятичного разделителя;
- направление операции задаётся только `type`; `amount` всегда положителен;
- дата обязательна, соответствует формату `YYYY-MM-DD` и является реальной
  календарной датой;
- комментарий обрезается по краям, пустое значение не сохраняется;
- название счёта обрезается по краям и не может быть пустым;
- дубли названий счетов после обрезки и без учёта регистра запрещены;
- будущая дата допустима;
- несуществующий `accountId` в загружаемом файле считается ошибкой валидации.

Ошибки ручного ввода показываются рядом с соответствующими полями. Ошибки
загрузки файла проходят через существующую панель ошибок и не приводят к
частичной загрузке данных.

## Расчёт итогов

Для каждой валюты независимо вычисляются:

```text
deposits = сумма amount для type = deposit
withdrawals = сумма amount для type = withdrawal
netFlow = deposits - withdrawals
```

Агрегаты учитывают все операции выбранной валюты, включая операции с будущей
датой. Выбранный счёт на агрегаты не влияет, поскольку фильтра по счёту в первой
версии нет.

## Поток данных и сохранение

1. Пользователь открывает форму, вводит или изменяет данные.
2. Форма нормализует значения и выполняет полевую валидацию.
3. Доменная функция иммутабельно создаёт обновлённый `PortfolioFile`.
4. `PortfolioContext` получает новый объект, после чего список и агрегаты
   пересчитываются из актуальных данных.
5. Как и остальные данные приложения, изменения записываются на диск после
   нажатия общей кнопки «Сохранить».

Удаление счёта обновляет справочник и все связанные транзакции одной операцией,
поэтому промежуточного состояния с невалидными ссылками не возникает.

## Тестирование

Автоматические тесты покрывают:

- загрузку старого файла и значения по умолчанию для новых массивов;
- валидацию допустимых и недопустимых транзакций и счетов;
- уникальность идентификаторов и ссылочную целостность;
- добавление, редактирование и удаление транзакций;
- сортировку по убыванию даты и порядок операций с одинаковой датой;
- фильтрацию RUB, USD и CNY;
- итоги пополнений, выводов и чистого потока отдельно по валютам;
- удаление счёта с переводом связанных операций в «Общий портфель»;
- нормализацию комментариев и названий, дубли счетов, будущие даты и дробные
  суммы.

Интерфейс вручную проверяется для создания, редактирования и удаления операций,
валютного фильтра, пустых состояний, подтверждений и мобильных карточек. В
проекте нет библиотеки компонентных тестов, поэтому её добавление только ради
этой функции не входит в объём первой версии.

## Критерии готовности

- пользователь может вести переносимую вместе с `portfolio.json` историю
  пополнений и выводов в RUB, USD и CNY;
- каждая операция имеет тип, положительную сумму, валюту и дату, а счёт и
  комментарий остаются необязательными;
- список автоматически отсортирован по дате по убыванию и фильтруется по
  валюте;
- агрегаты корректны и никогда не смешивают разные валюты;
- операции и ручные счета можно добавлять, изменять и удалять;
- удаление счёта сохраняет операции и переводит их в «Общий портфель»;
- старые файлы портфеля продолжают загружаться;
- новые доменные правила и файловая схема покрыты автоматическими тестами.
