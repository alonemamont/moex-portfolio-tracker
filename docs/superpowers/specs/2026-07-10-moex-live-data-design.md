# MOEX live data: замена мёртвого IMPORTRANGE на ISS API

Дата: 2026-07-10

## Контекст

`MICEX Index 99,5_.xlsx` — личная таблица учёта портфеля, отслеживающего индекс
МосБиржи (IMOEX). Изначально была Google Sheets, экспортирована в xlsx.
Тикеры/веса/цены/дивиденды/сектора тянулись через `IMPORTRANGE` из внешнего
Google-листа автора (`t.me/Finindie`). В экспортированном xlsx `IMPORTRANGE`
не работает — в ячейках только закэшированные значения на момент экспорта
(обёртка `IFERROR(__xludf.DUMMYFUNCTION(...), "закэш-значение")`).

Третий лист "Голубые фишки" (альтернативная стратегия) удалён по запросу
пользователя; `Main!C2`/`Main!H2` содержат неисправимые (по решению
пользователя) `#REF!`-фоллбэки на него — не в скоупе этой спеки.

## Цель

Сделать лист `Индекс Мосбиржи` полностью рабочим на актуальных данных,
без внешней Google-зависимости, при этом:
- не терять личные данные пользователя (доли/количество купленных акций)
  при обновлении рыночных данных;
- не публиковать личные финансовые данные в git-репозиторий.

## Источник данных

MOEX ISS API, бесплатный, без ключа, формат — **XML** (не JSON; JSON
использовался только на этапе разведки полей через `curl`):

- `https://iss.moex.com/iss/statistics/engines/stock/markets/index/analytics/IMOEX.xml?limit=100`
  → состав индекса: `ticker`, `shortnames`, `weight`. **Без `limit=100` ISS
  по умолчанию отдаёт только 20 строк (пагинация) — молчаливое обрезание
  списка.** Сейчас в составе 46 бумаг.
- `https://iss.moex.com/iss/engines/stock/markets/shares/boards/TQBR/securities.xml?securities=...&limit=100`
  → `LAST` (цена, фоллбэк `PREVPRICE` если торгов не было), `LOTSIZE`
  (лотность), `SHORTNAME`.
- `https://iss.moex.com/iss/securities/{SECID}/dividends.xml`
  → история выплат; берём последнюю запись `value` как "Дивиденд на акцию".
  Это **факт последней выплаты**, не forward-прогноз (в ISS прогнозов нет).
  Для бумаг без истории (недавнее IPO) — 0, не ошибка.

Поле `SECTORID` в ISS пустое у всех проверенных бумаг — официального
секторального классификатора на ISS нет. Сектор ведётся вручную.

## Компоненты

1. **`template.xlsm`** (git-tracked, чистый) — макро-версия текущего
   `.xlsx` (конвертация нужна для VBA). Все строки: `G`(Коэф-т)=1.0,
   `O`(Куплено)=0. Только live-колонки заполнены.
2. **VBA-модуль**, парсинг XML через `MSXML2.DOMDocument60` + XPath,
   без внешних библиотек/зависимостей.
3. **Кнопка "Обновить"** — тянет рыночные данные с MOEX.
4. **Кнопка "Загрузить позиции"** — тянет `G`/`O` из `positions.csv`.
5. **Кнопка "Сохранить позиции"** — пишет текущие `D`/`G`/`O` в `positions.csv`.
6. **Колонка `X` (новая, "Статус")** на листе `Индекс Мосбиржи` —
   `"в индексе"` / `"вне индекса"`. Колонка ранее не использовалась
   (проверено — пусто по всем 101 строкам).
7. **Справочник секторов** — `VLOOKUP`-таблица тикер→сектор на листе
   (или отдельном листе `Справочники`), заполняется вручную, не макросом.

## Маппинг колонок (лист `Индекс Мосбиржи`)

| Колонка | Источник | Правило обновления |
|---|---|---|
| `C`,`D` (тикер) | не трогаем | формулы как есть |
| `E` Название | ISS `shortnames`/`SHORTNAME` | live, всегда |
| `F` Вес | ISS `weight` | live если "в индексе"; **0** если "вне индекса" |
| `G` Коэф-т | — | ручной. Макрос "Обновить" никогда не трогает существующие строки. Новая строка → `1.0`. Меняется только "Загрузить позиции" |
| `H` Цена | ISS `LAST`/`PREVPRICE` | live, всегда (нужна для стоимости портфеля даже вне индекса) |
| `J` Лотность | ISS `LOTSIZE` | live, всегда |
| `O` Куплено | — | ручной, та же логика что `G` |
| `R` Сектор | справочник (`VLOOKUP`) | формула, не макрос |
| `S` Дивиденд | ISS `dividends`, последнее значение | live, всегда |
| `X` Статус (новая) | — | макрос: `"в индексе"` / `"вне индекса"` |

## Жизненный цикл строк (сопоставление по тикеру, не по номеру строки)

- Тикер есть и в таблице, и в свежем составе индекса → обновить
  `E,F,H,J,S`, `X`="в индексе".
- Тикер есть в таблице, но выпал из состава → `X`="вне индекса", `F`=0
  (не участвует в целевом распределении/`N`), но `H`,`S` всё равно
  обновляются (нужны для `P`, `T` — фактической стоимости и дохода
  позиции). `G`/`O` не трогаем.
- Тикер новый в составе, которого нет в таблице → новая строка в конец,
  `G`=1.0, `O`=0, `X`="в индексе".
- Дубли: перед вставкой новой строки — проверка `D:D` на совпадение
  тикера (регистронезависимо), при совпадении — пропуск.

## Фиксы формул под нулевой вес / динамический размер списка

- `N` (Соответствие): было `=M3/F3` → `#DIV/0!` при `F=0`. Меняем на
  `=IF(F3=0,"",M3/F3)`.
- `Main!K1`: было `=sum(N:N)/41` (захардкожен старый count) → меняем на
  `=AVERAGEIF('Индекс Мосбиржи'!X:X,"в индексе",'Индекс Мосбиржи'!N:N)`.
- `Main!M3` ("Соотв. MICEX"): было `=sum('Индекс Мосбиржи'!L3:L46)/C1`
  (диапазон захардкожен до строки 46) → `=sum('Индекс Мосбиржи'!L:L)/C1`,
  чтобы не резать данные при автодобавлении новых тикеров за пределы
  строки 46.

## Разделение личных данных от репозитория

Проблема: если рабочий `.xlsm` с реальными `G`/`O` закоммитить, любое
следующее сохранение после ввода личных данных попадёт в git-историю.

Решение:
- В git — только `template.xlsm` (чистый, `G`=1.0/`O`=0 везде) и исходники
  VBA-модулей (экспорт как `.bas`, для читаемых диффов).
- Рабочая копия пользователя (`MICEX Index 99,5_.xlsm`, реальное имя файла)
  — **никогда не коммитится**, в `.gitignore`.
- `positions.csv` (личные данные — доли и количество купленных акций) —
  локальный файл рядом с рабочим `.xlsm`, тоже в `.gitignore`. Формат:
  ```
  тикер,коэф,кол-во
  SBER,1.15,100
  GAZP,0.75,1540
  ```
- "Загрузить позиции" читает `positions.csv` из папки, где лежит текущий
  открытый файл (фиксированное имя, без диалога выбора). Тикер из CSV,
  которого нет в таблице — пропуск с предупреждением (`MsgBox`), новую
  строку не создаёт (это не задача кнопки позиций).
- "Сохранить позиции" перезаписывает `positions.csv` целиком на основе
  текущих `D`/`G`/`O` по всем строкам.

Рабочий цикл пользователя: клонировать репо → скопировать `template.xlsm`
в рабочий файл (вне git) → "Загрузить позиции" (если `positions.csv` уже
существует) → "Обновить".

## Обработка ошибок

- Нет сети / ISS недоступен: `MSXML2.ServerXMLHTTP`, таймаут 15с,
  `MsgBox` с текстом ошибки. Запись в ячейки — только после успешного
  получения **всех** трёх XML-ответов (всё или ничего).
- Тикер не торгуется/делистинг (нет строки в `securities.xml`): `H` не
  обновляется (остаётся последнее известное значение), `X`="вне индекса".
- Пустая история дивидендов: `S`=0, не ошибка.
- Пагинация ISS: все 3 запроса всегда с `limit=100`.

## Тестирование

- Ручной прогон "Обновить" на копии файла: строк со `X`="в индексе" = 46
  (текущий факт), сумма `F` по этим строкам ≈ 100%.
- Повторный запуск "Обновить" подряд — не создаёт дублей, не меняет
  `G`/`O`.
- Тест "вне индекса" (замокать состав без одного текущего тикера) —
  `N`, `Main!K1`, `Main!M3` не падают в `#DIV/0!`/`#REF!`.
- `scripts/recalc.py` (LibreOffice) не годится для проверки самого
  VBA-макроса (сеть/HTTP из VBA LO не выполняет так же) — используется
  только постфактум, чтобы убедиться, что формулы пересчитались без
  ошибок после ручного прогона макроса в реальном Excel.

## Вне скоупа

- `Main!C2`/`H2` `#REF!` на удалённый лист "Голубые фишки" — оставлено
  как есть по решению пользователя.
- Курс USD/RUB (`GOOGLEFINANCE`, `Main!M2`/`W1`) и котировка индекса
  через `IMPORTXML` (`Main!M1`) — отдельная задача, не затронуты.
